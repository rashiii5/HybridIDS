Project Report

HybridIDS

Hybrid VAE + LightGBM Intrusion Detection System with SHAP Explainability — NSL-KDD Benchmark

Network intrusion detection has become a critical component of modern cybersecurity infrastructure as organisations face increasingly sophisticated and diverse attack strategies that traditional signature-based systems fail to address. This project presents HybridIDS, a two-stage hybrid intrusion detection system that combines unsupervised anomaly detection with supervised attack classification and explainable AI to provide robust, transparent, and automated threat response on the NSL-KDD benchmark dataset.

HybridIDS is a two-stage hybrid intrusion detection system designed to improve the accuracy, robustness, and interpretability of network threat detection on the NSL-KDD benchmark dataset. As cyberattacks become increasingly sophisticated, traditional signature-based systems struggle to detect novel or zero-day threats. To address this limitation, HybridIDS integrates unsupervised anomaly detection with supervised classification and explainable AI in a unified pipeline.

The system first employs a Variational Autoencoder (VAE), trained exclusively on normal network traffic, to learn the underlying patterns of benign behaviour. By analysing reconstruction error, the VAE identifies anomalous connections that deviate from learned patterns, enabling detection of previously unseen attacks. These anomalies are then passed to a cost-sensitive LightGBM classifier, which categorises them into four major attack classes: Denial of Service (DoS), Probe, Remote-to-Local (R2L), and User-to-Root (U2R). To address class imbalance and improve detection of rare but critical attacks, the classifier is trained using SMOTE-ENN resampling and asymmetric cost weighting.

An ensemble thresholding strategy is used to accurately distinguish normal and anomalous traffic, while SHAP (SHapley Additive exPlanations) provides per-prediction feature importance along with plain-English explanations, enhancing transparency and analyst trust. The system is deployed using a FastAPI-based REST backend and a browser-based dashboard, allowing real-time monitoring and interaction. Additionally, a simulated firewall response module applies iptables-based defensive actions tailored to each detected attack type.

By combining unsupervised representation learning, cost-sensitive classification, and explainable AI, HybridIDS achieves improved performance on minority attack classes, better generalisation to unseen threats, and greater practical usability for cybersecurity analysts compared to standalone approaches.

Chapter 1

Introduction

1.1 Problem Statement

Existing machine learning-based intrusion detection systems suffer from three interconnected problems. First, purely supervised classifiers trained on imbalanced datasets systematically fail to detect rare but critical attack classes — particularly U2R (User-to-Root) — because the training signal from majority classes overwhelms the minority. Second, purely unsupervised anomaly detectors can flag that something is wrong but cannot identify the type of attack, which limits their operational utility. Third, black-box ML models provide no explanation for their decisions, making it impossible for security analysts to validate, trust, or act on the alerts they generate.

This project addresses all three problems within a single system: a hybrid two-stage architecture that uses a Variational Autoencoder for label-free anomaly detection, a cost-sensitive LightGBM classifier for attack categorisation, SMOTE-ENN and asymmetric class weights for imbalance handling, and SHAP explanations for transparency — evaluated on the NSL-KDD benchmark with a fully deployable REST API and analyst dashboard.

2.1 Dataset: NSL-KDD

NSL-KDD is a refined version of the original KDD Cup 1999 dataset, introduced by Tavallaee et al. to address the problems of duplicate records and biased class distributions that made KDD99 unsuitable for reliable evaluation. The dataset consists of 125,973 training samples and 22,544 test samples, each described by 41 features covering basic connection attributes (duration, protocol type, service, flag), content features (number of failed logins, root shell access), and traffic features (number of connections to the same host, error rates).

After one-hot encoding of three categorical features (protocol_type: 3 values, service: 66 values, flag: 11 values), the feature space expands to 122 dimensions.

2.2 Core Algorithmic Components

Variational Autoencoder (VAE)

A VAE is a generative deep learning model that learns a probabilistic latent representation of its input. Unlike a standard autoencoder which maps each input to a single point in latent space, a VAE maps each input to a distribution parameterised by mean z_mean and log variance z_log_var. The encoder produces these parameters; the decoder reconstructs the input from a sample drawn via the reparameterisation trick. Training minimises a combined loss of reconstruction error (MSE) and KL divergence (which regularises the latent space to approximate a standard normal distribution). For intrusion detection, the VAE is trained only on normal traffic. Attack traffic, lying outside the learned distribution, produces high reconstruction error at inference time.

LightGBM

LightGBM is a gradient boosting framework by Microsoft that builds an ensemble of decision trees using a leaf-wise growth strategy rather than the level-wise approach of XGBoost. It is highly efficient on tabular data, supports class-weight assignment for cost-sensitive learning, and is natively compatible with SHAP TreeExplainer. In this project, LightGBM receives a 130-dimensional input and produces probability estimates for each of the four attack classes.

SMOTE-ENN

SMOTE (Synthetic Minority Over-sampling Technique) generates synthetic samples for minority classes by interpolating between existing samples in feature space. ENN (Edited Nearest Neighbours) removes samples whose class label disagrees with the majority of their k nearest neighbours. SMOTE-ENN combines both — it oversamples minority classes and then cleans noisy boundary samples from the resulting distribution. This is particularly effective for intrusion detection because it both increases rare-class representation and removes ambiguous samples near class boundaries that would otherwise confuse the classifier.

SHAP (SHapley Additive exPlanations)

SHAP is a framework rooted in cooperative game theory that assigns each feature a contribution score (Shapley value) to a model's prediction for a specific input. For tree-based models like LightGBM, SHAP TreeExplainer computes exact Shapley values efficiently without approximation. The result for each prediction is a vector of feature contributions — positive values push toward the predicted class, negative values push away. This project uses these values to identify the top contributing features per prediction and generates plain-English summaries.

Chapter 3

Design

3.1 System Architecture

The system is structured as four horizontal layers. The Presentation layer contains all six HTML pages (index, dashboard, visualizations, comparison, explainability, firewall) which communicate with the API layer via HTTP fetch calls. The API layer is a FastAPI application exposing nine detection endpoints and six firewall endpoints, also serving static frontend files. The ML Pipeline layer contains five sequential modules — Preprocessor → VAE → Threshold Engine → LightGBM Classifier → SHAP Explainer — with model artefacts stored in backend/models/. The Data layer contains NSL-KDD raw files in data/raw/, processed NumPy arrays in data/processed/, evaluation results in results/, and action logs in logs.txt. The Firewall module sits alongside the API layer, receiving attack_type and IP from route handlers and issuing iptables commands to the operating system.

Chapter 5

Conclusion and Future Enhancement

This project presented HybridIDS, a hybrid intrusion detection system that successfully addresses three interrelated challenges in network security: rare-class detection failure, the opacity of black-box ML models, and the gap between research evaluation and deployable systems. By combining a Variational Autoencoder for unsupervised anomaly detection with a cost-sensitive LightGBM classifier for attack categorisation, SMOTE-ENN resampling and asymmetric class weights for imbalance handling, and SHAP-based explainability for transparency, the system demonstrated meaningful improvement over each of its constituent parts individually — most significantly, a 7× increase in U2R F1 score over a supervised-only baseline. The fully deployable FastAPI backend, six-page analyst dashboard, and simulated firewall response module demonstrate that the system bridges the gap between ML model development and operational security tooling.
