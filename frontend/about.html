Abstract

Traditional Intrusion Detection Systems (IDS) rely on supervised models that fail to generalise to novel attack variants and struggle with severe class imbalance — particularly for rare but critical attack categories such as User-to-Root (U2R) and Remote-to-Local (R2L).

This project proposes a hybrid two-stage framework that addresses both limitations. In the first stage, a Variational Autoencoder (VAE) trained exclusively on benign traffic learns the latent distribution of normal behaviour. Anomalies are detected by comparing log-transformed reconstruction error against an ensemble threshold derived from three complementary strategies. In the second stage, a cost-sensitive LightGBM classifier — trained on SMOTE-ENN balanced data with combined original and latent features — categorises detected anomalies into specific attack classes.

Evaluated on the NSL-KDD benchmark, the full hybrid system achieves a 7× improvement in U2R F1 over the supervised baseline, a Macro F1 of 0.694, and an MCC of 0.719, demonstrating the value of combining self-supervised representation learning with imbalance-aware classification.

Problem Statement

Existing IDS approaches face four core limitations:

  • Poor generalisation to unseen or zero-day attacks due to dependence on labeled attack data
  • Severe class imbalance — U2R has only 52 training samples vs 45,927 DoS samples
  • Anomaly-only detectors cannot classify attack type, reducing actionability
  • High false negative rates on rare, high-severity attack classes (U2R, R2L)

Key Contributions

  • VAE trained on normal traffic only — no attack labels required for anomaly detection stage
  • Ensemble threshold combining 95th-percentile, ROC-optimal, and F1-optimal methods
  • Combined feature space: original 122 features + 8-dimensional VAE latent vector (130 total)
  • SMOTE-ENN resampling balancing all four attack classes to ~45,900 samples each
  • Cost-sensitive weights (12× for U2R) in LightGBM to further prioritise rare classes
  • SHAP explainability with plain-English summaries for every flagged sample
  • Full-stack web interface with live detection, visualisations, and model comparison

Methodology

Phase 1 — VAE Training

The VAE encoder maps 122-dimensional input to an 8-dimensional latent space via Dense(64) → Dense(32) → Dense(16) → [z_mean, z_log_var]. The decoder reconstructs the original input from a sampled z. Training uses only normal traffic samples with a combined MSE + KL divergence loss. BatchNormalization and EarlyStopping (patience=8) are applied.

Phase 2 — Ensemble Thresholding

Reconstruction errors are log-transformed (log1p) to handle extreme outliers. Three thresholds are derived: 95th percentile of normal errors (weight 0.25), Youden's J optimal from ROC curve (weight 0.40), and F1-maximising sweep (weight 0.35). The final threshold is their weighted average: 0.3224.

Phase 3 — SMOTE-ENN + Classification

Attack samples are balanced using SMOTE-ENN, bringing all four classes to ~45,900 samples. The classifier input concatenates the 122 original features with the 8-dimensional z_mean from the VAE encoder (130 features total). LightGBM is trained with cost-sensitive sample weights: DoS×1, Probe×1.5, R2L×4, U2R×12.
